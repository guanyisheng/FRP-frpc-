# FRP 客户端配置生成器 (frpc)

可视化生成 [frp](https://github.com/fatedier/frp) 客户端 `frpc.toml` 配置文件，无需手写 TOML。单文件 HTML，浏览器打开即用，数据仅在本地处理。

## 功能概览

| 模块 | 说明 |
|------|------|
| 向导 · 选择模式 | 导入已有 `frpc.toml`，或从零新建 |
| 向导 · 服务器连接 | 地址、端口、协议、Token、STUN、高级传输 |
| 添加代理 | TCP / UDP / HTTP / HTTPS / STCP / P2P 快速卡片 |
| 代理列表 | 表格查看、编辑、删除、Ping 测延迟 |
| 配置文件 | 实时预览完整 `frpc.toml` |
| 保存配置 | 右上角弹窗：预览、一键复制、下载 |

---

## 详细使用流程

### 阶段一：打开工具

1. 用 Chrome / Edge / Safari 等现代浏览器打开 `index.html`（或 `FRP客户端配置文件自动生成工具.html`）。
2. 页面顶部为标题 **frpc**，下方为两步向导；完成向导后会出现顶部导航与右上角 **保存配置** 按钮。

---

### 阶段二：① 选择模式

**路径：** 首页 → 步骤条「① 选择模式」

#### 方式 A：导入已有配置

1. 点击卡片 **「导入已有配置」**（左侧，带 📥 图标）。
2. 下方出现文本框，将完整的 `frpc.toml` 粘贴进去（需包含 `serverAddr`、`[[proxies]]` 等）。
3. 点击 **「下一步 →」**。
4. 工具会自动解析：
   - 服务器地址、端口、Token、传输协议、STUN 等 → 填入步骤②
   - 所有 `[[proxies]]` 段 → 填入后续「添加代理 / 代理列表」

#### 方式 B：从头开始

1. 点击卡片 **「从头开始」**（右侧，带 ✨ 图标）。
2. 无需粘贴内容，直接点击 **「下一步 →」**。
3. 在步骤②中手动填写服务器信息，再逐步添加代理。

> 未选择模式就点「下一步」会提示：请先选择创建模式。

---

### 阶段三：② 服务器连接

**路径：** 步骤条「② 服务器连接」

#### 基础项（默认展开）

| 字段 | 说明 | 示例 |
|------|------|------|
| 服务器地址 | frps 公网 IP 或域名 | `154.219.114.11` |
| 端口 | frps 监听端口 | `7000` |
| 协议 | 客户端与服务器通信方式 | `TCP`（常用）、QUIC、KCP、WebSocket、WSS |
| 认证令牌 | 与服务端 `auth.token` 一致，可留空 | 复杂字符串 |
| STUN 服务器 | P2P / XTCP 穿透用，可留空 | 默认 `stun.easyvoip.com:3478` |

#### 高级传输配置（点击折叠条展开）

| 字段 | 默认值 | 说明 |
|------|--------|------|
| TCP 多路复用 (tcpMux) | 勾选 | 减少连接数 |
| 心跳间隔 | 120 秒 | `transport.heartbeatInterval` |
| 心跳超时 | 90 秒 | `transport.heartbeatTimeout` |
| TCP Mux 保活间隔 | 30 秒 | `transport.tcpMuxKeepaliveInterval` |

#### 完成本阶段

1. 确认 **服务器地址** 已填写（必填）。
2. 点击 **「完成，进入配置 →」** 进入主界面。
3. 需要修改服务器时：顶部导航点 **「服务器」** 可回到本步骤。

---

### 阶段四：添加代理

**路径：** 顶部导航 → **「添加代理」**

#### 添加一条代理

1. 在 **「快速添加代理」** 区域点击协议卡片：

   | 卡片 | 类型 | 典型用途 |
   |------|------|----------|
   | TCP | 端口转发 | SSH、RDP、游戏服务器 |
   | UDP | UDP 映射 | 游戏 UDP、远程桌面 UDP |
   | HTTP | 反向代理 | 内网网站、面板 |
   | HTTPS | 加密反向代理 | HTTPS 服务 |
   | STCP | 秘密 TCP | 需访问端 visitor 配合 |
   | P2P | XTCP 点对点 | NAT 穿透，需 STUN |

2. 弹出 **添加 XX 代理** 窗口，填写：

   - **名称**：自动生成（如 `http_80_a3k9x2`），无需手填，不支持中文。
   - **本地地址**：内网服务 IP，默认 `127.0.0.1`。
   - **本地端口**：
     - 点击输入框展开 **预设端口列表**（按协议分类，含 SSH/HTTP/游戏等说明）。
     - 可 **多选** 多个端口（每条端口生成一条代理）。
     - 也可 **手动输入** 端口号，按回车添加。
   - **HTTP / HTTPS 额外项**：
     - **域名方式**（单选）：**自定义域名** 或 **子域名**。
     - 自定义域名：如 `app.example.com`，多个用英文逗号分隔。
     - 子域名：如 `web01`（需 frps 配置子域名后缀）。
   - **TCP / UDP 额外项**：远程端口，留空则自动为「本地端口 + 1000」。
   - **STCP / P2P 额外项**：`secretKey`，留空自动生成。

3. 点击 **「添加」** 关闭弹窗，代理出现在 **「已添加」** 列表中。

#### 管理已添加代理

- **编辑**：列表右侧「编辑」→ 修改后保存。
- **删除**：「删除」→ 确认后移除。

---

### 阶段五：代理列表（可选）

**路径：** 顶部导航 → **「代理列表」**

1. 以表格查看所有代理：类型、名称、本地地址、远程端口/域名。
2. **Ping**：对配置了 HTTP/HTTPS **自定义域名** 的条目测延迟（显示 ms 或失败原因）。
3. **全部 Ping**：依次检测所有可 Ping 的代理。
4. 同样支持 **编辑**、**删除**。

> Ping 在浏览器内发起请求，仅适用于 HTTP/HTTPS 且已填域名的代理；纯 TCP/UDP 无域名时显示「无域名」。

---

### 阶段六：查看 / 导出配置

有两种方式，内容一致。

#### 方式 A：配置文件页

**路径：** 顶部导航 → **「配置文件」**

- 全屏显示当前生成的 `frpc.toml` 全文。
- 增删代理或修改服务器后内容 **自动更新**。
- **双击** 文本区域可快速复制到剪贴板。

#### 方式 B：保存配置（推荐）

**路径：** 右上角 **「保存配置」**（完成向导后出现）

1. 点击 **「保存配置」**。
2. 弹窗显示完整 `frpc.toml` 预览。
3. 选择操作：
   - **一键复制** → 粘贴到记事本，保存为 `frpc.toml`。
   - **下载文件** → 直接得到 `frpc.toml`。

---

### 阶段七：在服务器上使用

1. 将 `frpc.toml` 上传到运行 frpc 的机器。
2. 启动客户端：

```bash
chmod +x frpc
./frpc -c /path/to/frpc.toml
```

3. 在 frps 服务端确认隧道已连接；HTTP/HTTPS 需保证域名已解析到 frps 公网 IP。

```

```

## 部署到 GitHub

### 私有仓库 + 不公开访问

| 方案 | 条件 | 访问范围 |
|------|------|----------|
| **私有 Pages** | GitHub Pro / 团队版 | 仅仓库成员登录后可访问 |
| **仅 clone 本地用** | 免费私有仓库 | 不开启 Pages，协作者本地打开 HTML |
| **公开 Pages** | 公开仓库 | 任何人可访问 URL，勿填真实 Token |

**推荐步骤（私有 Pages）：**

1. 创建 **Private** 仓库，上传文件：
   - `index.html`（由工具 HTML 重命名）
   - `README.md`
2. **Settings → Pages**：Source 选 `main` 分支，文件夹 `/ (root)`。
3. **Settings → Pages → Visibility**：选 **Private**。
4. 将 Pages 地址发给协作者（需对方有仓库权限）。

**公开 Pages（需注意安全）：**

1. 公开仓库 + 开启 Pages。
2. 访问 `https://<用户名>.github.io/<仓库名>/`。
3. 演示环境勿填写生产环境 Token。

## 常见问题

**Q：导入后代理数量为 0？**  
检查粘贴内容是否包含 `[[proxies]]` 段，且格式为标准 TOML。

**Q：保存配置按钮看不到？**  
需先完成向导两步并点击「完成，进入配置」。

**Q：HTTP 访问不了？**  
确认 `customDomains` 域名已 DNS 解析到 frps IP，且 frps 已配置 `vhostHTTPPort` 等。

**Q：名称能否中文？**  
不能，自动生成格式为 `协议_端口_随机字符`，仅英文与数字。
